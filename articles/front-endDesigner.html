Front-end Designer

Geschreven door Linda van Dijk Op 25 juni 2016

Er zijn veel verschillende beroepsrol benamingen binnen de web industrie. Zo heeft ieder bedrijf zijn eigen benamingen voor bepaalde rollen. Dit kan soms verwarrend werken. De meest voorkomende namen zijn:

  • Web Designer
  • Front-end Developer
  • UI Designer
  • UX Designer
  • Interaction Designer
  • Art Director
  • Web Developer
  • Full Stack Developer
  • Content Strategist

De huidige beroepsrollen komen steeds dichter bij elkaar. Soms design je in HTML en CSS omdat je dat direct ziet wat er gebeurt met een responsive design. Ook interacties ontwerp je vaak direct in HTML en CSS omdat je dan direct het resultaat ziet en kan aanvoelen of de interactie natuurlijk is en past in het ontwerp. Doordat verschillende beroepsrollen worden uitgevoerd door één persoon ontwikkelen zich nieuwe benamingen.

Front-end Designer

Ik noem mijzelf een Front-end Designer, een naam die je steeds vaker hoort maar nog niet bij iedereen bekend is. Een Front-end Designer zit ergens tussen Design, UX en Development in. Een Front-end Designer creëert visual design met HTML, CSS en minimalistisch JavaScript. Overeenkomende namen voor deze combinatie zijn:

  • UI Developer
  • Client-side Developer
  • UI Engineer
  • Design Engineer
  • Frontend Architect
  • Designer/Developer
  • Prototyper

Multifunctioneel

Als Front-end Designer ben je in veel markten thuis. Je moet overal wat van af weten maar niet tot in detail. Je bent een perfecte tussenpersoon in een project en dat wordt steeds belangrijker. Front-end Designers:

  • Begrijpen UX principes, maar spenderen geen tijd aan onderzoek, flows schetsen en scenario’s bedenken/ schetsen.
  • Hebben oog voor design, maar zijn niet bezig met het uitzoeken van lettertypes die bij elkaar passen, kleurenpaletten en illustraties en iconen maken.
  • Kunnen Javascript schrijven, maar kunnen niet hele applicatie code schrijven en debuggen.
  • Begrijpen hoe belangrijk Back-end Development is en hoe het ongeveer in elkaar steekt, maar schrijven geen Back-end logica, servers en testscripts etc.

Het kan natuurlijk zo zijn dat niet alle punten gelden voor iedereen en dat je wel iconen maakt en kleurenpaletten samenstelt. Mijn advies is, doe wat je leuk vindt en klamp je niet vast aan één bepaalde beroepsrol. Wees duidelijk in wat je wel en niet wilt en ga ook zeker geen dingen doen die je niet leuk vindt.

De gebruiker

Als Front-ender moet je rekening houden met de gebruiker. Het zijn niet alleen designers die ervoor moeten zorgen dat de UX van een website goed is, slechte code lijdt ook tot slechte UX voor de gebruiker. Bedenk maar eens als een website langer dan 5 seconde moet laden dan ben je negen van de tien keer al weg. Het is van belang om te zorgen voor goede performance. Zorg ervoor dat plaatje niet onnodig groot zijn en dat je semantische HTML, CSS en Javascript schrijft. Bedenkt ook dat je te maken hebt met veel verschillende gebruikers die jouw website op verschillende browsers en op verschillende devices bekijken. Ook heb je te maken met gebruikers die geen muis gebruiken maar alleen een toetsenbord of kleurenblind zijn. Je hebt te maken met extreem lage netwerkverbindingen en met gebruikers die opzettelijk uitzetten elementen uitzetten zoals Javascript, custom fonts, localStorage en/of cookies. Het is aan jou als Front-ender om daar rekening mee te houden en daar fallbacks voor de schrijven zodat iedereen jouw website kan gebruiken.

Daarnaast lijdt slechte code ook tot slechte UX voor Developers. Als je geen gestructureerde code schrijft, kost het meer moeite voor Developers om met jouw code te werken. Daarom is het van belang om te weten hoe de Back-end ongeveer in elkaar steekt zodat je de code zo schrijft dat de Back-ender er direct mee verder kan en niet eerst jouw code moet verbeteren/ veranderen.

Conclusie

Er ontstaan steeds nieuwe beroepsrollen in de web industrie doordat er steeds nieuwe technologieën ontstaan. De Front-end Designer wordt steeds belangrijker binnen een project omdat hij ervoor zorgt dat de gebruiker een goede ervaring beleefd en dat iedere gebruiker goed om kan gaan met het interface van de website of app. Daarnaast is de Front-end Designer een handige schakel binnen een project omdat hij overal wel wat vanaf weet. Dus weet je nog niet precies wat je wilt worden? Word Front-end Designer!
