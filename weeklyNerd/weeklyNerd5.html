Browser bloat

Peter-Paul Koch (PPK)

Dinsdag 15 maart 2016


Tools lossen de problemen van het web niet op, zij zijn het probleem. Dit is waar de hele presentatie van Peter-Paul Koch om draaiden. Er zijn tegenwoordig zoveel tools en libraries die je eenvoudig kunt implementeren om je website. Maar eigenlijk wordt je website daar alleen maar slechter van. De ene library heeft een andere library nodig en voor je het weet heb je een hele lap code die je gebruiker moet inladen als zij jouw website bezoeken. De gebruiker is dus uiteindelijk de dupe van onze luiheid. Want zijn MB’s gaan daardoor hard achteruit.

Nadelen JavaScript libraries
  • Gebrek aan controle:
    De versie van de libraries kan altijd geüpdatet worden. In het ergste geval wordt de library verwijderd van de CDN en is jouw hele website kapot.
  • Stabiliteit:
    Met externe libraries zal je website zal niet alleen uit de lucht zijn als jouw webserver uit de lucht is, maar ook als de externe library host uit de lucht is.
  • Privacy:
    Je gaat uiteraard respectvol om met de privacygegevens van je bezoekers. Maar je weet nooit wat de hosting service met de privacygegevens van jouw gebruikers doet.
  • Veiligheid:
    Als de externe hosting kwaad in de zin heeft, kunnen zij alles met jouw website doen.

Het zijn voornamelijk de laatste twee punten die het risicovol maken. De voordelen zijn eigenlijk maar klein en zeker niet de moeite waard om externe partijen controle over je website te geven. (Bron)

Een echt goede front-ender schrijft zoveel mogelijk JavaScript zelf en gebruikt alleen libraries die hij begrijpt en kan aanpassen. Hou het zo eenvoudig mogelijk en zorg dat je geen overbodige code in je website hebt.

JavaScript is nadelig voor mobiele bezoekers. Denk je dat de mobiele bezoeker de 10.000 lijnen van de JavaScript code die jij toegevoegd (of gekoppeld) hebt in de pagina zal uitvoeren? Ja waarschijnlijk wel, maar je bezoekers zal niet wachten als het te lang duurt. (Bron)
